TLDR wa-1.3 -> wa-2.0: реализация основана теперь на flexbox, а не на float. Дополнительный класс-модификатор .fields.form более не требуется и не поддерживается. Также более не действует класс .value.no-shift. Вместо него отступ сверху задается для элемента .name. А внутри одного .field несколько .value теперь работают иначе: выстраиваются по горизонтали, а не по вертикали.

Компонент .fields используется для отображения структурированных данных в формате «ключ — значение» и любых форм ввода данных.

.fields в wa-2.0.css сохранил свою внутреннюю семантику по сравнению с wa-1.3.css, однако реализация полностью обновлена. Теперь никаких float: left. Все основано на жестком display: table;.

Важное отличие .fields в wa-2.0.css от wa-1.3.css состоит в том, в обновленной реализации расположение нескольких .value подряд внутри одного .field выстроят их в один горизонтальный, а не вертикальный ряд. Для вертикального отображения необходимо реализовывать оформление контента в виде списков ul > li внутри одного .value (стандартное оформление ul внутри .field > .value в этой связи отключено).

Внутри .fields каждый элемент списка (формы) оформляется в виде элемент .field с дочерними элементами .name и .value.

С помощью элементов .fields-group можно выделить несколько элементов .field в единую логически и визуально обособленную группу. Группировку рекомендуется применять во всех сложных формах настройки.

Между элементами с классом .field допускается использование заголовков класса .heading.

.field > .value.submit добавляет отступ слева для расположения кнопки по одной вертикальной линии с элементами input и позволяет обойтись без добавления пустого элемента .name.

Дополнительный класс на общий элемент .fields.vertical позволяет разместить весь компонент .fields в одну узкую колонку. В этом случае .fields будет выглядеть, как на мобильных устройствах. Полезно для формата отображения в сайдбаре (узкой боковой колонке).

Дополнительный класс на элемент .field.vertical размещает поле в одну колонку. Полезно при размещении последнего поля с широкой таблицей или большой редактируемой областью.

Отступ для .name

Вариаций контента, располагающегося в .value, невообразимое множество. Поэтому неизбежно, что .name может быть визуально слегка не выровненным в соответствии с контентом .value.

Но в связи с тем, что основных типов контента внутри .value встречается не так много — просто текст, input> для ввода текста, textarea или select, кнопка, чекбокс/радиокнопка — для выравнивания контента .name в зависимости от содержимого .value предусмотрены следующие классы-модификаторы:

  • .name (без дополнительных классов): если в .value какой-то обычный статический текст,
  • .name.for-input: если в .value элемент ввода данных: input, textarea, select,
  • .name.for-button: если в .value кнопка .button,
  • .name.for-checkbox: если в .value чекбокс или радиокнопка .wa-*,
  • .name.for-switch: если в .value .waSwitch() .switch,
  • .name.middle и .value.middle: для вертикального центрирования содержимого ячеек.
