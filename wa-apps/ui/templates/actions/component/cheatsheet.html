Шпаргалка 1.3 → 2.0

Краткий список того, с чем почти наверняка столкнется разработчик приложения Webasyst при проверке своего приложения в новом окружении wa-2.0.css:

  1. Скорее всего, «поползет» общий лейаут. Это из-за перехода #wa-app, .sidebar и .content на display: flex;. Попробуйте для начала поставить на проблемный блок (контейнер) display: block;. А затем переводите .sidebar / .content в своем приложении на режим flexbox.
  2. .menu-v надо заменить на .menu.
  3. .menu-h надо заменить на .chips.
  4. Иконки .icon10 и .icon16 нужно заменить на векторные иконки-глифы Font Awesome либо на свои — с классом .icon. Старые спрайты более не поддерживаются.
  5. .userpic20 нужно заменить на .userpic.
  6. Класса .block больше нет, так как все отступы заложены в основных элементах оформления и контента (внутри разных меню и т. д.), чтобы не возникало недопонимание, какие отступы где нужно делать. Теперь их и не надо делать — они уже есть в каждом компоненте! Если все же нужно создать блок с произвольным контентом, используйте .box. Для общего блока .content — либо .article, либо .box.contentbox.
  7. .left200px и .left300px у сайдбара нужно заменить на .width-XX согласно документации.
  8. .inline-link больше нет! Можно просто убирать весь код a.inline-link > b > i и оставлять только a.
  9. Ссылки синего цвета остались только в контентной области (в текстах). В навигационных элементах они теперь не отличаются по цвету от основного контента (черные, белые). Если нужно выделить ссылку синим цветом, то можно воспользоваться классом .link.
  10. Вообще теперь все цвета нужно декларировать в виде CSS custom variables внутри var(), и не прописывать их хардкодом. Иначе темный режим, который нельзя отключить в wa-2.0.css, будет отображать интерфейс неправильно. Да-да, не забывайте, что теперь поддерживается темный режим!
  11. Шрифтовая пара Helvetica Neue / Lucida Grande более не используется. Теперь везде в интерфейсах используется полностью нативный для ОС пользователя набор шрифтов.
  12. Модификаторы .small, .large, .smaller, .larger, .smallest, .largest работают для большинства интерфейсных элементов. Однако на некоторых компонентах (например, на .menu) отключены сознательно для уменьшения возможности «сбить ритм» и целостное восприятие интерфейса.
  13. .indicator переименован в .badge.
  14. .profile больше нет. Используйте .flexbox или .list.
  15. Спиннер загрузки .icon16.loading больше не поддерживается ни в каком виде. Для индикатора процесса загрузки контента используйте .skeleton (рекомендуется), .waLoading() или .spinner.
  16. Подключать wa-content/js/jquery-wa/wa.core.js, wa-content/js/jquery-wa/wa.dialog.js, wa-content/js/dialog/dialog.js больше не нужно.
    Для работы всех JS-компонентов дизайн-системы Webasyst 2 подключайте один единственный файл — wa-content/js/jquery-wa/wa.js.
  17. Использование старого класса .errormsg нужно заменить на новые .state-error или .state-caution в зависимости от ситуации.
  18. Классы .float-left и .float-right больше не работают — вместо них нужно использовать другую верстку, например, с применением класса .flexbox.
  19. Если в JavaScript-коде продукта используются вызовы .waDialog(), придется переписать их с учетом новых требований и обновить HTML-верстку диалогов.
  20. Список будет пополняться.
