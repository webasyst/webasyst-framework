Размер таблицы можно корректировать с помощью классов table.smallest — table.largest:

{_render class="small"}

table.bigdata

Таблицы с большим количеством числовых данных рекомендуется оформлять в интерфейсе с помощью table.bigdata. Добавление класса незначительно уменьшит отступы и размеры шрифтов внутри таблицы: чуть меньше стандартного вида таблицы, но чуть крупнее table.small:

{_render class="bigdata"}

td

Выравнивание контента внутри ячеек регулируется с помощью классов td.align-left (по умолчанию), td.align-center, td.align-right и td.valign-top, td.valign-middle (по умолчанию), td.valign-bottom.

Форсированная установка минимальной и максимальной ширины для ячейки — td.min-width и td.max-width.

tr.selected

Если в таблице используются чекбоксы, выбранный ряд рекомендуется выделять с помощью tr.selected.

table.zebra

table.bordered

Таблица, в которой у каждой ячейки есть явные границы table.bordered.

{_render class="bordered"}

table.borderless

Таблица без визуальных границ у ячеек table.borderless.

{_render class="borderless"}

table.single-lined

Таблица с фиксированной высотой каждой строки и обрезанием внутреннего контента в случае, когда он не помещается, — table.single-lined. Что-то вроде ellipsis, только с градиентным обрезанием контента.

Вариант с .single-lined можно совмещать со всеми перечисленными выше вариантами таблиц: .zebra, .bordered, .borderless.
