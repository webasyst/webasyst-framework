wa-2.0.css предоставляет множество служебных классов, которые не поддаются общей классификации, но будут полезны при точечном использовании на конкретных элементах. Большинство этих классов пришло в дизайн-систему из предыдущего поколения (wa-1.3.css) и сохранено в wa-2.0.css в силу зарекомендовавшей себя полезности, накопленного объема кода и, отчасти, привычки.

Выравнивание:

  • .align-left — выравнивание текста по левому краю (text-align: left),
  • .align-left-mobile — то же самое, но только на мобильных устройствах,
  • .align-center — выравнивание текста по центру (text-align: center),
  • .align-center-mobile — то же самое, но только на мобильных устройствах,
  • .align-right — выравнивание текста по правому краю (text-align: right),
  • .align-right-mobile — то же самое, но только на мобильных устройствах.
  • .valign-top — вертикальное выравнивание к вершине блока (vertical-align: top),
  • .valign-middle — вертикальное выравнивание по центру блока (vertical-align: middle),
  • .valign-bottom — вертикальное выравнивание к нижней границе блока (vertical-align: bottom).

Типографика:

  • .bold — жирное начертание (font-weight: bold),
  • .semibold — полужирное начертание (font-weight: 500),
  • .italic — курсив (font-style: italic),
  • .underline — подчеркивание текста (text-decoration: underline),
  • .strike — зачеркнутый текст (text-decoration: line-through),
  • .highlighted — для подсветки части текста или даже целого блока «маркером»: .highlighted, .highlighted.green, .highlighted.pink, .highlighted.blue, .highlighted.orange.
  • .black — контрастное выделение ярким темным цветом (в светлой теме) и ярким светлым (в темной) — хорошо сочетается с жирным начертанием,
  • .link — выделение отдельного текста в виде контентной ссылки (синим — в светлой теме и желтым — в темной),
  • .gray — блеклый вспомогательный текст серого цвета,
  • .hint — небольшая подсказка такого же серого цвета, только меньшего размера,
  • .state-error,
    .state-success,
    .state-caution — способ отобразить сообщение о блокирующем, успешном и ошибочном состоянии в рамках текстового блока без специального фона (например, когда использование .alert не уместно),
  • .state-error-hint,
    .state-success-hint,
    .state-caution-hint — подсказки к блокирующему, успешному и ошибочному состоянию элементов формы ввода данных,
  • .editable — подсветка интерактивного элемента по :hover,
  • .uppercase — верхний регистр (text-transform: uppercase),
  • .lowercase — нижний регистр (text-transform: lowercase),
  • .capitalize — каждое слово с заглавной буквы (text-transform: capitalize),
  • .nowrap — неразрывный текст (white-space: nowrap),
  • .break-all — слова в тексте могут быть разбиты в любом месте, лишь бы поместиться в контейнер (word-break: break-all),
  • .break-word — разбиение текста будет выполняться с сохранением целых слов (word-break: break-word).

  • .text-ellipsis — текст будет размещаться в одну строку до максимальной заданной ширины, а все, что не помещается, будет обрезаться многоточием.

Блоки и изображения:

  • .blank — делает блок или контейнер «чистым листом» (белым в светлом режиме, близким к черному — в темном),
  • .not-blank — заполнение чуть «приглушенным» базовым фоном var(--background-color),
  • .dark-mode-inverted — в темном режиме содержимое блока или изображения будет инвертировано (mix-blend-mode: screen).

Размеры:

  • .smallest — 50%,
  • .smaller — 75%,
  • .small — 87.5%,
  • .large — 125%,
  • .larger — 150%,
  • .largest — 200%.

Перечисленные классы для корректирования размера предназначены только для текстового контента (span, p, strong, b, i, a) и некоторых изолированных атомарных интерфейсных элементов вроде полей для текстового ввода и кнопок (input, button). Для цельных навигационных блоков типа .sidebar, .menu, .tabs, .fields их применение не предусмотрено с целью сохранения единого стройного вида интерфейса и уменьшения визуальной энтропии.

Для удобства пользователя элементы ввода (input, textarea, select) при назначении классов-модификаторов размера .smallest — .largest имеют меньший диапазон: 75%—150% вместо 50%—200%.

Видимость в зависимости от устройства пользователя:

  • .hidden — элемент скрыт на всех устройствах (display: none),
  • .mobile-only — виден только на телефонах (display: block при ширине экрана <= 760px),
  • .tablet-only — только на планшетах (display: block при ширине экрана от 761px до 1024px),
  • .desktop-only — только на десктопах (display: block при ширине экрана >= 1025px),
  • .desktop-and-tablet-only — на планшетах и десктопах, но не на телефонах.
