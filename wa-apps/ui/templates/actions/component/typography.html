Базовое семейство шрифтов определяется операционной системой пользователя и позволяет приблизить визуальное восприятие приложений Webasyst к большинству приложений, установленных на устройстве пользователя.

font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen-Sans, Ubuntu, Cantarell, "Helvetica Neue", sans-serif;

Базовый размер элемента явно не задается и, следовательно, определяется браузером. Это значит, что в подавляющем большинстве случаев будет использоваться 1rem = 16px, хотя пользователь может переопределить это значение в настройках браузера.

Для отдельных элементов и целых навигационных блоков размеры шрифтов могут меняться, и везде масштаб задается относительно — в rem (не в px!). Масштаб зависит от типа устройства пользователя и функционального назначения элемента. Например, для заголовка h1 размера шрифта = 2.5rem (40px). Для меню .menu — 0.875rem (14px). Масштаб в единицах rem изначально рассчитан исходя из кратности стандартного значения 1rem = 16px:

     1px = 0.0625rem
     2px = 0.125rem
     3px = 0.1875rem
     4px = 0.25rem
     5px = 0.3125rem
     6px = 0.375rem
     7px = 0.4375rem
     8px = 0.5rem
     9px = 0.5625rem
    10px = 0.625rem
    11px = 0.6875rem
    12px = 0.75rem
    13px = 0.8125rem
    14px = 0.875rem
    15px = 0.9375rem
    16px = 1rem

В реальности размер 1rem будет зависеть от предпочтений пользователя (настроек браузера), но в подавляющем большинстве случаев будут использоваться именно эти размеры, поэтому рекомендуется придерживаться указанных выше значений rem.

Предположение о стандартном размере 1rem = 16px позволяет использовать иконки Font Awesome именно в том размере (в той сетке), для которого они изначально были созданы. Для элементов интерфейса с измененным шрифтом размеры иконок Font Awesome (<svg>) нормируются к 16px (например, в .menu), чтобы иконки не масштабировались без особой необходимости.
