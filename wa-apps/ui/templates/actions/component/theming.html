wa-2.0.css официально поддерживает два режима работы интерфейса бекенда: светлый (по умолчанию) и темный. Режим влияет только на цветовые решения и никак не затрагивает навигацию, структуру и поведение интерфейса. На геометрию интерфейсных элементов цветовой режим бекенда также не влияет.

Реализация режимов работы интерфейса происходит не на этапе сборки CSS, а в runtime — с помощью CSS custom properties (CSS variables). Это позволяет в реальном времени включать и менять режим при изменении соответствующего режима на устройстве пользователя. По умолчанию светлый и темный режимы включаются автоматически в зависимости от окружения — @media (prefers-color-scheme: dark).

Поддержка светлого и темного режимов на уровне приложения реализуется просто — вместо явного указания значения цвета необходимо использовать соответствующую переменную из wa-2.0.css или своего собственного CSS-файла. Например, вместо { color: #333; } использовать { color: var(--text-color-strong); }.

* * *

Поддержки каких-либо прочих режимов помимо светлого и темного на системном уровне Webasyst не планируется, чтобы не создавать излишнюю «головную боль» разработчикам приложений.
